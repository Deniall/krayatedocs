kRAYate Engine Documentation

Welcome to the kRAYate Engine Documentation.

In case you don't know what kRAYate is, visit this link: kRAYate Game Engine.

So you've just downloaded the BaseGame.zip, extracted, ran the .exe, and now want to make your own game. Great!

Unfortunately, due to only being a short project, we did not get the time to create a fully functioning editor to work with the engine. However, you can still use this engine to make your own game, using your own graphics, animations, sprites and scripts! Let's go through gameData.ini first of all.

General game parameters

[Game]
Tells us these are the general game parameters.
name
The name you want your game to have!
texturesUsed
The number of textures in the /Data/Textures folder.
noOfLevels
The number of levels you wish your game to have.
playerSpeed
The walking speed of the player character.
playerRotate
The rotation speed of the camera/player.
screenWidth
The width of the game window in pixels.
screenHeight
The height of the game window in pixels.

Level parameters

[Level1]
Tells us what level these parameters are for. Add one of these for every level you create.
texture
Path of the level .png file. (Explained fully below)
sprites
Path of the levels sprites .csv file. (Explained fully below)
playerX
The X-coordinate of the players location when they spawn in the level. (32 by default)
playerY
The Y-coordinate of the players location when they spawn in the level. (32 by default)
ceiling
The texture that you wish the ceiling to be. (Explained fully below)
floor
The texture that you wish the ceiling to be. (Explained fully below)

Prefab parameters

[Prefab1]
A prefab is a set of parameters given to an object in the game. Add one for every object you wish to add.
baseName
The name of the object that the prefab parameters will apply to.
tex
The texture you wish your object to have if not animated.
animated
Can be 'true' or 'false'. True if you wish it to have an animated texture, False if not.
fps
The number of times it should update the texture per second.
animation
The texture numbers, comma delimited, of the textures in the animation.
solid
If 'true', you cannot pass through the object. If false, you can.
radius
The radius of the collision box for the object.
script
Any scripts you wish to add to the object.
loadLevel
Given a level number, it will load that level on collision with the object. If set to 'none', it will not.
